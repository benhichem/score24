## Common Development Commands

* **Install dependencies**
  ```bash
  bun install
  ```
* **Run the application**
  ```bash
  bun run index.ts   # equivalent to `bun --hot ./index.ts`
  ```
* **Start the server with hot‑module‑replacement**
  ```bash
  bun --hot ./index.ts
  ```
* **Run the test suite**
  ```bash
  bun test                # runs all `*.test.ts` files
  bun test path/to/file.test.ts   # run a single test file
  ```
* **Build a static asset** (HTML, TS/TSX, CSS)
  ```bash
  bun build index.html   # bundles the entry point and its dependencies
  bun build frontend.tsx
  bun build index.css
  ```
* **Lint / format** (no built‑in script; use `bunx` to invoke a linter you add, e.g.)

## High‑Level Architecture

- **Runtime** – The project runs on **Bun** (v1.x). All commands should use the Bun CLI (`bun`, `bunx`, `bun test`, etc.).
- **Server entry point** – `index.ts` creates an HTTP server via **`Bun.serve()`**. Routes are declared in a plain object; the root (`"/"`) serves the HTML file `index.html`. An API route (`/api/users/:id`) returns JSON. Optional WebSocket support is configured under the `websocket` key.
- **Frontend** – `index.html` imports a React component (`frontend.tsx`) using a `<script type="module">` tag. Bun’s built‑in bundler transpiles JSX/TSX on‑the‑fly, so no separate bundler (Webpack/Vite) is required.
- **React UI** – The UI lives in `frontend.tsx`. It uses **React 18** (`createRoot`) and can import CSS directly (`import './index.css'`).
- **Static assets** – Any `.html`, `.tsx`, `.jsx`, `.js`, or `.css` file referenced from the HTML can be imported directly; Bun bundles them automatically.
- **Project layout**
  - `index.ts` – server configuration and routing.
  - `frontend.tsx` – React component rendered on the page.
  - `index.html` – HTML entry point that loads the React bundle.
  - `src/` – currently empty; intended for future TypeScript source files.
